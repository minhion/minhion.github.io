# v21 Issue Reporting


Welcome to v21 Beta Issue Reporting!



This Git Project is intended for any issues that may arise while testing the v21 beta installation and its features. All issues should be reproduceable and reported within Git's Issue functionality.

> _**Note:** All issues **must** be reported within this Git Project. Please refrain from using email or other communication methods to report any issues or bugs._


## Instructions

With this project, we are introducing a **new method** for reporting issues utilizing **Description Templates**. Please follow the below steps to properly report your issue: 


- Select **[this link](https://ctm-git.trybmc.com/v21-beta-testing/v21-issue-reporting/-/issues/new?issue%5btitle%5d=&issuable_template=default)**.
    You will be presented a default template (Use this if no other template is appropriate)
- Provide a descriptive **Title** to your issue
- Validate that the **Type** selected is **Issue**
- Select the appropriate template:
    - **Configuration** - this includes issues for **Agent**, **Plug-Ins** & **Centralized Connection Profiles**
    - **MFT** - This includes issues for **MFT** & **MFTe**
    - **Monitoring** - This includes issues within the Monitoring Domain
    - **Planning** - This includes issues within the Planning Domain
    - **Tools** - This includes issues for **Calendars, Conditions, Controls Resources, Shared Variables, Site Standards, User Views, Reports & Application Integrator** 
    - **Default** - Used for all other issues
   <br> ![Choose Template Image](../images/v21issues-ChooseTemplate.png)
   <br>
- For **Configuration, MFT & Tools** select the appropriate sub-category by marking an X within the corresponding square brackets
    <br> ![Choose Sub-Category](../images/v21issues-SubCategory.png)
    <br>
- Complete the **"Issue Summary"** section
- Complete the **"Steps to Reproduce"** section
- Select **"Create Issue"** button
- _**Done!**_


> _**Note:** The section below "## PLEASE DO NOT MODIFY BELOW HERE" is used to assign the issue and label to your issue so you do not need to assign or add labels._


## What's next?

After you've reported an issue, an appropriate team member will attempt to reproduce the issue. If the issue is unable to be reproduced, a team member will reach out to you for additional instructions. 
<br>
<br>
Once issues are successfully reproduced, data is collated and reported to R&D on a weekly basis.
